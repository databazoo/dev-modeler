UC14. I want to program in PL/SQL

This UC describes how one can store application logics in database itself. It is assumed a project is already set (See UC01).

PL/SQL is generally supported by SQL inputs. Particular options depend on target database type.

To create a function or procedure (if supported by database type):

  1. You can Right-click a schema on canvas or in object tree.
  2. Choose “Add new function” option.
  3. Fill in “Name”, “Arguments” and other fields according to selected database type.
  4. Fill in “Returns” with primitive type, TRIGGER (for triggered functions), TABLE (for tabular results) or leave empty (to create procedures). See your database documentation for complete syntax reference.
  5. Fill in “Function body” in PL/SQL (or other language available for your database type).
  6. check the generated SQL in “SQL” tab at the right side of the window.
  7. Press “Create” or hit “Ctrl+S”.

To create a package (if supported by database type):

  1. Right-click a schema on canvas or in object tree.
  2. Choose “Add new package” option.
  3. Fill in “Package name”, “Package definition” and other fields according to selected database type.
  4. Fill in “Package body” in PL/SQL (or other language available for your database type).
  5. You can check the generated SQL in “SQL” tab at the right side of the window.
  6. Press “Create” or hit “Ctrl+S”.

To create a trigger (if supported by database type):

  1. Right-click a table on canvas or in object tree.
  2. Choose “Add new trigger” option. An existing trigger function is required in PostgreSQL for this option to be available (see UC14.a on how to create one).
  3. Fill in “Trigger name”, “Events” and other fields according to selected database type.
  4. Fill in “Trigger body” in PL/SQL (or other language available for your database type) or choose a trigger function to execute.
  5. You can check the generated SQL in “SQL” tab at the right side of the window.
  6. Press “Create” or hit “Ctrl+S”.
