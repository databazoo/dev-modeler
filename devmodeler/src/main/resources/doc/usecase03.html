UC03. I want to set up a complete development stack

This UC shows how to set up a complete dev stack - set up a new project and connect it to Version Control System. It is assumed that you are familiar with VCS you are going to use (like GIT, SVN, etc). For simplicity, this example will use GIT commands, but feel free to adapt them to VCS of your choice.

  1. Checkout your VCS project into local folder (git clone)
  2. Create a subfolder called “db” and add it to VCS (git add)
  3. In “db” create a subfolder called “changes” and add it to VCS (git add)
  4. Go to menu File - Projects.
  5. Select “Create a new project” or press “New” button.
  6. Switch to advanced mode.
  7. Select database type and press “Next”.
  8. Choose appropriate server count.
  9. Fill in connection information for every environment listed in “Server connections” table. Use “Delete” key to remove unneeded rows.
  10. Press “Next”.
  11. If access information is correct (for the first environment) a list of available databases will be shown. Choose one or more and press OK.
  12. Fill in “Project name”.
  13. Set “Store revisions to” path to point to “your project folder / db / changes”. Changes you make will be stored there in XML format.
  14. Press “Create project” button.
  15. You may want to use menu Model - Rearrange items to arrange downloaded objects automatically on the canvas. Try different options for better results and finetune placements manually.
  16. Go to menu File - Projects.
  17. Press “Export”.
  18. Choose the project configuration to be saved in “your project folder / db / config.dproj”. This way any developer who checks out your project can import your database stack information.
  19. Add “your project folder / db / config.dproj” to VCS (git add).
  20. Commit your project to VCS (git commit, git push)
  21. If you are using Enterprise Edition you may want to use menu File - Projects and choose Versioning tab to set up automatic operation of VCS.

NOTE: “Host” input fields may contain full resource information including port (:5432), Oracle SID (:SID or /serviceName), MSSQL instance (\instance), etc. See JDBC connection string documentation for your target DB type.
