UC10. I want to make changes and publish them

This UC describes how to make both structural and data changes, pack them into revisions and publish them. It is assumed a full development stack is set (See UC03).

  1. Make structural changes:
    1. Create table “customers” (See UC05).
    2. Add columns (id, firstname, lastname) and a primary key (See UC05).
    3. Select “Add new table” in the tree.
    4. Fill in “Name” with “customer_types”.
    5. Press “Save” or hit “Ctrl+S”.
    6. Add columns (id, title) and a primary key (See UC05).
    7. Close the wizard.
    8. Select “customers” table.
    9. Drag column “id” to table “customer_types” column “id”. A 1:N wizard will open.
    10. Set “Local column” to “New column”.
    11. Set “New column name” to “customer_type_id”.
    12. Press “Save” or hit “Ctrl+S”.
  2. Fill in data:
    1. Fill in at least one row into “customer_types” (See UC06).
    2. Fill in at least one row into “customers” (See UC06).
    3. Press “SQL query” hot button at the top of the main window.
    4. Insert UPDATE customers SET customer_type_id = 1 into SQL input.
    5. Press “Run query” or hit “F5”.
    6. Press “Add to revision” to add your manual UPDATE to revision.
    7. Choose last outgoing revision in “Select revision” select and press “Add to revision”.
    8. Close data windows.
  3. Commit the change to VCS:
    1. The change XML file is already on disk, just commit it (git commit, git push).
  4. Publish the change:
    1. Switch view using “Changes” button at the top of the main window.
    2. Double-click your revision (last outgoing revision).
    3. Click “Apply” to Production (or other environment).
    4. Review the query.
    5. Click “Apply to Production”.
    6. Close the wizard window.
  5. Commit to VCS:
    1. The change XML file was updated. Commit your change (git commit, git push).
