UC04. I want to share my configuration with other developers

This UC describes how to share project or model configuration. It is assumed a project is already set (See UC01).

To share a complete project (models, environments, etc.):
  1. Go to menu File - Projects.
  2. Press “Export”.
  3. Choose where your project configuration to be saved.
  4. Add the configuration file to VCS or send it to other developers.

On target workstation:

  1. Download the configuration file.
  2. Go to menu File - Projects.
  3. Press “Import”.
  4. Choose the configuration file.
  5. Confirmation window appears if project with same name already exists. Choose a different project name or merge the configuration.
  6. Optionally, open the project and press “Sync” at the top of the main window to synchronize models with physical databases.
To share the model of a single database:
  1. Go to menu Model - Export - To XML.
  2. Choose where your model will be saved.
  3. Add the configuration XML file to VCS or send it to other developers.

On target workstation:

  1. Download the configuration file.
  2. Open the target project and database.
  3. Go to menu Model - Import.
  4. Choose the configuration XML file.
  5. Press “Sync” at the top of the main window to synchronize models with physical databases.

NOTE: passwords are not stored in XML export files. Use encrypted .dproj export files if you want to store passwords, too.
