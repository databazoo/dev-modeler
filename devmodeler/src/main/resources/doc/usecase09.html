UC09. I want to optimize my model

This UC describes how one can optimize the database schema. It is assumed a project is already set (See UC01).

  1. Switch view using “Optimizer view” button at the top of the main window.
  2. Choose “Static analysis” tab to check for missing indexes, index clashes, useless tables, etc.
  3. Choose “Data-based analysis” tab to check for wasteful column definitions and simplification options.
  4. Choose filtering options and press “Analyze”.
  5. Go through individual lines to find out on design flaws.

NOTE: the database should be filled with real-world data to show decent results from data-based analysis.

NOTE: some flaws can not be detected for sure. Optimizer view only shows suggestions and potential problems. It is the designer who is to decide what to do about it.
