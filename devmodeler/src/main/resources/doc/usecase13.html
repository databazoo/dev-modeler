UC13. I want to migrate my schema to a different DB type

This UC describes how one can migrate schemata between different database types. It is assumed the 2 projects with different types are already set (See UC01).

  1. Open source project.
  2. Go to menu Model - Export - To SQL.
  3. Choose appropriate dialect (target database type).
  4. Press “Export” and save the export file.
  5. Go to menu Model - Export - To XML.
  6. Press “Export” and save the schema layout.
  7. Open target project.
  8. Press “SQL query” hot button at the top of the main window.
  9. Insert contents of the export file into SQL input.
  10. Run the query. If (for any reason) the query should fail, run individual statements by selecting them in the SQL input and pressing “Run” or hitting “F5”.
  11. Close the SQL window.
  12. Go to menu Model - Import.
  13. Choose previously exported schema layout file and press “Import”.
  14. Press “Sync” at the top of the main window to synchronize the model with the physical database.

NOTE: Keep in mind that database types are not fully compatible. Not all schema properties can be moved between database types due to different support for datatypes, objects, trigger events, etc.
