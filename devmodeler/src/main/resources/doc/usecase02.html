UC02. I want to create an empty database or model

This UC describes how one can connect to create an empty database, creating a new project on the way. It is assumed that you have the rights to create a database on target server.

  1. Go to menu File - Projects.
  2. Select “Create a new project” or press “New” button.
  3. In simple mode:
    1. Fill in “Project name”.
    2. Select database type or “Abstract model”.
    3. Fill in “Host”, “User” (optional) and “Password” (optional).
    4. Fill in “Database” with a desired database name.
    5. Press “Create” button.
    6. Confirmation of creation will appear. Press “OK”.
  4. In advanced mode:
    1. Select database type or “Abstract model” and press “Next”.
    2. Choose appropriate server count.
    3. Fill in connection information for every environment listed in “Server connections” table. Use “Delete” key to remove unneeded rows.
    4. Press “Next”.
    5. Fill in “Project name”.
    6. Choose “Store revisions to” path. Changes you make will be stored there in XML format. If you already use VCS for your application source files you can choose your database changes to share same location.
    7. Fill in a desired database name into “Databases” table.
    8. Press “Create project” button.
    9. Confirmation of database creation will appear. Press “OK”.
    10. Databases on other environments will be created upon first communication with the target server. Confirmation of creation will appear. Press “OK”.

NOTE: “Host” input fields may contain full resource information including port (:5432), Oracle SID (:SID or /serviceName), MSSQL instance (\instance), etc. See JDBC connection string documentation for your target DB type.
